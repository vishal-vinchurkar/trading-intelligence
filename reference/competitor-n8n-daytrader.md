## 1. Architecture (structured from the transcript)

- **Interface:** Telegram — send a ticker in, get the trade opinion back. Push/mobile/conversational.
- **Orchestration:** n8n (low-code visual workflow).
- **Data point 1 — multi-timeframe intraday candles** via **twelvedata.com** (free tier: 800 calls/mo, 8/min):
  - 1-minute × last 100 candles
  - 15-minute × last 100 candles
  - 1-hour × last 100 candles
  - Merge (3 inputs) → Aggregate to one item → optional Code node (JS) to relabel/clean the ~300 data points so the LLM ingests them more easily.
- **Data point 2 — news sentiment** via **newsapi.org** (free):
  - Query `q=<TICKER>`, `from=today-1day` → last 24h of articles.
  - OpenAI **GPT-4.1-mini** node reads articles → JSON `{category: pos/neutral/neg, score, rationale}`.
- **Merge** candles + sentiment → Aggregate to one item.
- **Final AI agent** (OpenAI **GPT-4.1**, "smartest that doesn't break the bank like o1/o3"):
  - Prompt: "expert day trader… single unified recommendation buy/sell/hold."
  - Instructed to **group candles by timeframe, calculate its OWN indicators (RSI, MACD, trendlines), confirm trend with the 1h data**, then decide.
  - System message forces output to exactly: `technical recommendation`, `entry price`, `stop-loss`, `target/exit price` — no rationale, no commentary.
- **Output → Telegram** (send text to chat ID).
- Pitch: *"everything is data focused, so there's no bias at all."* Human decides whether to take the trade.

Example shown: Tesla → SELL, entry 325, stop above, target/exit 322. Apple → HOLD (all fields N/A).

## 2. Comparison vs our system

### What he does that we don't
| His move | Real gap for us? | Verdict |
|---|---|---|
| News / sentiment layer (24h articles → LLM sentiment) | **Yes — genuine hole.** We have zero catalyst awareness. | Steal *selectively* (as a risk-flag overlay, not core signal). |
| Telegram push delivery | Yes (UX) | Worth adding (Phase 4 delivery). Delivery, not intelligence. |
| 1m/15m/1h multi-timeframe | Yes, but it's a **day-trading** style | Mismatched to our 5/15/30-**day** horizon; intraday is high-noise + not free for us. **Skip.** |
| n8n low-code | Faster to assemble | Doesn't improve output. Irrelevant to quality. |

### Where ours is materially stronger
1. **He has no backtest. None.** "No bias, data-focused" is marketing — he never measures if the opinion is right. We have a **10-year point-in-time, out-of-sample backtest** with per-band hit-rate + alpha. This is the whole ballgame and it's lopsidedly ours. His is a *polished* version of the "science project."
2. **"AI calculates its own indicators"** = LLM doing arithmetic over 300 candles in-context → unreliable, non-reproducible, hallucination-prone, **un-backtestable**. Ours computes RSI/MACD/ATR **deterministically in Python** — identical every run, auditable, backtestable.
3. **He outputs an opinion; we output a placeable trade** — entry / 2×ATR stop / structural target / **computed R:R gated at 1.5**.
4. **Volatility-scaled expected moves** (sized to realized σ) — he has none.
5. **Calibrated conviction** — our "confidence" *is* the backtested hit-rate; his is the model's feeling.

> The "no bias" claim is the trap: an LLM opining from numbers + recent news is *full* of bias (training priors, recency of the headline it just read, prompt framing). Real bias-removal = a deterministic score validated out-of-sample — the part he lacks and we have.

### Verdict
Ours is the materially superior **decision engine**. His is superior only on two **peripheral** things: a **news/event layer** (a real hole in ours) and a **push delivery channel** (UX). His core "AI reads candles + news and opines" is *less* defensible than ours. Nothing in his setup should change the quant core — if anything it validates that the backtest is our moat.

## 3. Decisions — incorporate vs reject

**Incorporate (folds into Phase B / Phase 4):**
- ✅ **Event/catalyst awareness** — earnings dates (free via yfinance) + an "elevated headline risk" flag. Add as a **risk-flag overlay**, NOT a backtested core signal (news has no clean point-in-time free history — same honesty rule as fundamentals: keep it out of the score).
- ✅ **Telegram push** of the daily Top Signals — low effort, high "front-and-centre" value, matches the watchlist ask. Phase 4 delivery.

**Reject:**
- ❌ **Intraday 1m/15m/1h multi-timeframe** — wrong horizon, high noise, not free for us.
- ❌ **LLM-computed indicators** — strictly worse than our deterministic spine; breaks backtestability.
- ❌ **Single-agent merge** — we keep agent isolation + the quant spine.

**APIs he uses (for reference if we add the news layer):** `twelvedata.com` (intraday candles, free 800/mo), `newsapi.org` (news, free). We'd only adopt newsapi-style news for the *flag*, and we already use yfinance for price.
